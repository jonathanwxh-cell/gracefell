## #43 — unit tests for the pure functions
Added **Vitest** (`npm test` → `vitest run`) and `src/game/engine.test.ts` (**14 tests**): the math
helpers (`clamp`/`lerp`/`dist`/`angTo`/`angDiff`), deterministic `seededRandom`, the
**`difficultyForGrace` balance contract** (the published −3..+5 table — MEASURED/JOURNEY/Oath rows +
out-of-range clamping), and the **`isVictoryScore` save-schema v4 validator**. To make
`difficultyForGrace` testable without a canvas, its logic was extracted into a **pure exported
module function**; the `Game.difficultyForGrace()` method now delegates to it, so the single
derivation point is preserved. `isVictoryScore` was exported. CI runs `npm test` before the
Playwright gate.

## #42 — removed the `as any` on the render methods
The 10 render methods (`render`, `drawArena`, `drawHUD`, …) were assigned as
`(Game.prototype as any).X = function …`. An `export interface Game { render(): void; … }` merge
already declares them, so the `as any` was **redundant** — all 10 are now plain `Game.prototype.X`
assignments and `strict` is no longer defeated for that block. The full render-extraction refactor
was deliberately **not** attempted (high risk, low benefit on a live 3.9k-line engine).
