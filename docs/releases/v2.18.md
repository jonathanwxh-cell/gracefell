# Gracefell v2.18 — adaptive phase score

This release is based on `origin/main` through the parallel v2.16–v2.17.1
server, dependency, web-polish, and code-quality stream. The rebase preserved
those changes and added a Windows-safe immutable-cache check for `/audio/` and
`/assets/`.

Date: 2026-07-24  
Implementation: Codex (GPT-5 / Codex Desktop)  
Music generation: MiniMax Music 3.0  
Status: local implementation and acceptance complete; not committed, pushed, or deployed in this pass

## Outcome

Gracefell now has one related, background-safe cue for each boss phase:

1. Quiet Ash
2. The Sovereign Burns
3. Gracefall

The score changes phase in under one second, remains behind combat, survives
pause/resume and playback failure, and does not grow its Web Audio graph across
retries. Players can separately set Music and Combat effects from an in-fight
MIX dialog that freezes simulation while keeping the score audible.

## Fixed by Codex

### Music selection and mastering

The user kept the first Music 3.0 choice and accepted replacement choices for
Phases 2 and 3. The API outputs were audition material, not shippable loops.
Read-only music-direction review caught:

- a long Phase 2 opening dropout;
- Phase 2 and Phase 3 fade-shaped loop resets;
- a Phase 3 introduction too quiet for phone speakers;
- a later Phase 3 surge that could mask the busiest combat;
- an obsolete 4.82 MB score still entering the production bundle.

Codex re-cut the accepted sources into stable, exact 78 BPM forms and built a
permanent one-second wrap crossfade into each master. The old single score was
removed.

### Streaming and phase control

`GameAudio` now owns two permanent streaming decks. The inactive deck changes
`src`; its media element, media source, and gain are reused. This corrected an
early implementation that retained disconnected nodes after repeated phase
loads.

- Phase quantization: nearest upcoming 78 BPM beat only when it is within
  250 ms; otherwise transition immediately.
- Crossfade: 720 ms equal-power curve.
- Reset: Phase 1 is requested immediately.
- Retry: an incoming playback rejection retains the requested phase and retries
  twice.
- Recovery: an outgoing or incoming resume failure restores and restarts the
  outgoing deck.
- Fallback: persistent streaming failure selects the procedural score.
- Pause: the AudioContext clock suspends during an actual pause, so an active
  crossfade resumes from the same point.
- Performance: Phase 1 is eager; later phases request metadata only until used.
- Procedural bed: scheduler timing now derives from the same 78 BPM constant.

The established route and mix ceilings remain:

`deck -> presence dip -> low-pass -> soundtrack -> music -> master`

- SFX submix: 1.00
- music bus ceiling: 0.24
- soundtrack submix ceiling: 0.56
- soundtrack presence dip: -6 dB at 1.8 kHz

### Strongest-wins action ducking

Music ducking now retains a set of expiring requests. The strongest active
request owns the music gain, so a later light swing cannot raise the score
during a critical boss warning. Requests use a 15 ms attack and 120 ms
recovery. The same envelope survives a user volume change and recovers fully
after the last request expires.

### MIX and persisted controls

Save schema v5 adds:

- `musicVolume` — default 0.85
- `sfxVolume` — default 1.00

Legacy saves migrate to those values. The in-fight MIX dialog:

- freezes combat time, AI, and player input;
- deliberately leaves the score clock running for live adjustment;
- provides Music and Combat effects sliders plus TEST SFX;
- keeps focus inside the modal;
- removes the canvas and PAUSE control from the active tab order;
- consumes Done, Escape, P, and backdrop input;
- clears combat buffers and returns focus to the canvas on close;
- is disabled while the player owns a manual PAUSE.

This separates a live mix audition from a true pause: manual PAUSE still
suspends both simulation and audio.

## Review loop

Three independent read-only lanes reviewed the pass:

- music direction measured loop seams, loudness, silence, spectral balance, and
  phase arrival;
- adaptive-audio engineering probed rejected playback, pause during crossfade,
  outgoing restoration, procedural fallback, and 60 repeated deck swaps;
- audiovisual playtesting exercised desktop and 390×844 MIX, pause ownership,
  dismissal, touch targets, action leakage, and phase arrival.

The initial implementation was not accepted. Review findings caused two
mastering recuts, a sub-second transition ceiling, permanent deck reuse,
playback recovery, live MIX auditioning, explicit dismissal, manual-pause
exclusion, and modal focus containment. Final music-direction and audio-engine
reviews signed off with no remaining blocker.

## Performance and scope

- Production audio payload fell from one 4.82 MB score to three cues totaling
  2.81 MB.
- The score remains streamed; it is not decoded into full in-memory
  `AudioBuffer`s on mobile.
- No boss health, attack damage, difficulty curve, player timing, touch-action
  geometry, camera rule, scoring rule, or visual rendering budget changed.
- `SOUNDTRACK_VERSION` is `2.18`, preventing older CDN/browser music from being
  reused after a future deployment.
- The local generation workbench and API responses remain untracked development
  material. No API key or expiring download URL enters the repository or bundle.
