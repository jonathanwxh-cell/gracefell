# GRACEFELL v2.12.1 release record

## Outcome

GRACEFELL v2.12.1 fixes the only player-facing defect that survived a three-person all-path
playtest and applies a small, performance-neutral combat-clarity pass.

- Receive Grace cannot be accepted by movement buffered before the lethal hit. It requires a
  fresh post-death key press or a deliberate tap on the visible offer.
- Oath I/II retain their authored two-beat packets but introduce them with less raw speed, damage,
  and recovery compression.
- Journey tells and Oath packet progress use one larger backed combat chip.
- Two small spirit arcs separate the player from Malakar only at close overlap.
- The mobile ATK circle shows whether one or two light follow-ups have been accepted into the
  existing queue.
- A record-heavy 390px title gives victories, best time, and last grade a compact dedicated line.

No runtime asset, texture, sprite, particle system, save migration, boss phase, attack packet,
player damage, collision rule, touch target, mobile reaction floor, music track, or SFX identity
changed.

### Rejected reports

The review did not treat every automated observation as a defect:

- A reported three-click path skip was disproved by direct production pointer input: +4 moved
  through +3 and +2 to +1 exactly once per click.
- A wrong path after reload came from several same-origin reviewer tabs sharing localStorage.
  Isolated persistence passed.
- A quarter-scale renderer appeared only after debug-assisted state manipulation and did not
  reproduce in natural play.
- Missing `window.__game` was limited to individual reviewer browser edges. Direct production and
  the release suite exposed the hook normally.

## Confirmed root cause and fix

Movement keys place a short TTL entry in `Input.pressed`. A lethal hit can transition the game
from `fight` to `dead` before that TTL expires. `handleDeathGraceInput()` previously consumed
`left` without distinguishing whether the press belonged to combat or the terminal screen.

`onPlayerDeath()` now consumes the carried left press as part of changing input ownership. This
preserves the visible left-arrow keyboard shortcut but requires the player to release and press
again after death.

The new true-touch regression records:

```text
after lethal transition: state dead, grace 0, carried left false
after one terminal frame: state dead, grace 0
after fresh left press:   state dead, grace -1
```

The same suite then taps the offer, proves it owns the current confirmation sequence, and verifies
that a later fresh tap still starts the retry.

## Presentation contract

### Combat cue chip

Named beginner tells and Oath packet counts are mutually exclusive by design. One centered chip
therefore owns that transient information without stacking panels. It uses existing Canvas text
and rectangles, larger phone type, a dark backing, and a spirit or ember border.

### Overlap separation

When the player enters Malakar's close-contact radius, two short spirit arcs appear around the
player. They do not alter collision, targeting, i-frames, or hitboxes and disappear outside the
overlap. This is one small Canvas path, not a new particle or post-process.

### Queued touch input

The existing light-only queue may hold two follow-ups. The ATK circle now draws one or two small
diamonds for those accepted inputs. This is not combo progress: the arena's
`CHAIN 1/3 → 2/3 → LIGHT FINISHER` counter still advances only on connected hits.

### Record-heavy title

At widths below 520px, the saved-run line uses 12px type, a 36px horizontal inset, and a dedicated
position above the start prompt. A synthetic 390×844 acceptance case with 12 victories, best 1:23,
and last grade S measured:

- saved-run line: y 485.3, rendered width 170.2px, maximum width 354px;
- start prompt: y 506.4;
- touch controls sentence: y 540.2.

## Changed from v2.12

- Buffered pre-death left movement no longer accepts Grace.
- Oath I/II raw pressure is eased without changing their packet rules.
- Journey tells and Oath counters gain a backed, larger combat chip.
- Close overlap gains a player separation rim.
- Accepted queued light inputs gain touch-button feedback.
- Record-heavy narrow titles gain a dedicated compact line.
- Package version advances from 2.12.0 to 2.12.1.
- Save schema v4, first-run Journey, tutorial persistence, lethal-source advice, Oath III–V,
  scoring, boss phases/patterns, touch geometry, collision, music, SFX, and runtime assets remain
  unchanged.
