## #45 — server security headers
`server.mjs` now sends `X-Content-Type-Options: nosniff`, `Referrer-Policy: no-referrer`, and
`X-Frame-Options: DENY` on every response (files, `/health`) via a shared `SECURITY` object. A full
`Content-Security-Policy` is **deliberately deferred** until the display fonts are self-hosted (#39)
— a CSP today would have to whitelist the Google Fonts CDN, so it waits for that pass. Verified
live that none of these were present before (even through Cloudflare); present now.

## #46 — MP3 served `immutable`
The 4.6 MB score lives under `/audio/`, which `server.mjs` only ever tagged `no-cache` (`immutable`
was `/assets/`-only). The audio URL is already `?v=SOUNDTRACK_VERSION`-versioned, so `immutable` is
safe: `const immutable = path.startsWith('/assets/') || path.startsWith('/audio/')`. Returning
visitors now cache the score for a year instead of revalidating 4.6 MB per load.

## #38 — removed `kimi-plugin-inspect-react`
The opaque dev-only Babel Vite plugin (personal npm account, mirrored from a private GitLab) is gone
from `vite.config.ts` and `package.json`; the lockfile was regenerated. The production build is
unaffected — the bundle is ~0.85 KB smaller without the injected source-location attributes.

## #41 — adopted `npm ci` (the "broken lockfile" claim was stale)
Reproduction showed `npm ci` installs cleanly (exit 0) from the committed lockfile — the
README/AGENTS "npm ci fails — use install" note was **outdated**. Corrected: README, AGENTS,
`info.md`, and the CI workflow (`.github/workflows/qa.yml`) now use `npm ci` for deterministic
installs, which is exactly the reproducibility #41 asked for. Verified again after the #38 lockfile
regen (clean `node_modules` → `npm ci` → exit 0).

## #44 — LICENSE file
Added `LICENSE` ("All rights reserved", matching the README's stated intent) and set
`package.json` `"license": "UNLICENSED"`. Removes GitHub's ambiguous "No license" badge.

## QA / verification
A new `qa/verify.cjs` assertion fetches the QA server and asserts `nosniff` + `Referrer-Policy` on
the root and `immutable` on the `/audio/` MP3. Direct-curl confirmed all four headers and the mp3
`immutable` (assets still immutable, html still `no-cache`). `npm run lint` and `npm run build`
