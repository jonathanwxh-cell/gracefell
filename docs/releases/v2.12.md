# GRACEFELL v2.12 release record

## Outcome

GRACEFELL now has a clearer path from a first completion to expert mastery.

- A true first-time player starts on **JOURNEY -2**, a disclosed recommended path with the same
  sovereign, phases, attacks, health, and records, but longer reads, softer wounds, and four
  flasks.
- Existing saves retain their selected path. A legacy save with no stored path remains on
  **MEASURED 0**.
- One contextual rite teaches movement, rolling through danger, perfect-dodge poise damage, and
  punishing a stagger. Completion persists.
- Death identifies the lethal attack and gives one actionable response. From attempt two onward,
  the player may explicitly **RECEIVE GRACE** without accidentally triggering the retry behind it.
- Positive paths are disclosed as **Oaths**. They retain the existing speed, damage, flask, poise,
  and stagger curve while adding authored boss follow-up packets and shorter recovery.
- Connected light hits now show `CHAIN 1/3`, `CHAIN 2/3`, and `LIGHT FINISHER` in the fight. A miss,
  heavy attack, roll, or wound breaks the visible chain.

No phase checkpoint, hidden dynamic difficulty, smaller touch target, shorter mobile telegraph
floor, new runtime asset, or per-frame environment effect was added.

## Design basis

The design separates assistance from mastery rather than treating difficulty as one opaque
multiplier.

- [Xbox Accessibility Guideline 108](https://learn.microsoft.com/en-us/xbox/accessibility/xbox-accessibility-guidelines/108)
  supports descriptive difficulty choices and meaningful reduced-challenge options.
- [Xbox Accessibility Guideline 103](https://learn.microsoft.com/en-us/xbox/accessibility/xbox-accessibility-guidelines/103)
  supports redundant visual/audio channels for critical information.
- [Hades' official FAQ](https://www.supergiantgames.com/blog/hades-faq/) separates explicit,
  death-responsive assistance from Hell Mode and the Pact.
- [Hades' Welcome to Hell notes](https://www.supergiantgames.com/blog/hades-welcome-to-hell-update-patch-notes/)
  demonstrate expert rules that change encounter decisions, not only health and damage.
- [Steelrising's developer account](https://news.xbox.com/en-us/2022/09/08/steelrisings-assist-mode-feature/)
  describes souls-like assistance aimed at specific barriers.
- [Dead Cells Update 29](https://dead-cells.com/patchnotes/29) shows both granular assistance and
  the state complexity introduced by continuation systems.

The resulting rule is: make GRACEFELL easier to **finish** without making it easier to **master**.
Assistance remains visible and voluntary; expert challenge remains stable, learnable, and recorded.

## Contextual teaching and recovery

The rite occupies one short, screen-space strip and advances only through performed actions:

1. move;
2. roll through the blade;
3. learn that a perfect dodge damages poise;
4. strike the actually staggered boss.

The next beat replaces the previous one; the game does not display a tutorial wall. Once the
stagger is punished, `tutorialComplete` is persisted and the rite does not return.

Every damaging boss path now records its source: swipe, slam, charge, volley, meteor, ring, or
spiral. The death screen maps that source to a single next-attempt instruction. `unknown` retains
a safe generic fallback.

From attempt two, **Receive Grace** moves the selected path down by exactly one step, never below
-3. It is an offer, not an automatic balance change. Touching the offer owns that confirmation
sequence, so the same gesture cannot restart behind the selection. The ordinary touchscreen and
pointer-only **rise again** paths remain intact.

## Oath boss packets

The Oath system does not invent new unreadable hazards. It composes four already-authored direct
attacks into capped routes:

- volley → charge → swipe;
- charge → swipe → slam;
- swipe → slam → volley;
- slam → volley → charge.

Ring, meteor, and spiral are excluded from packets to avoid stacking area denial. An attack must
still satisfy its normal cooldown, phase, and range eligibility before it can become the next
step. Immediate repeats receive a lower selection weight.

Oath I/II can schedule one follow-up. Oath III/IV use the same capped length with greater
frequency and recovery pressure. Oath V can schedule two follow-ups. The HUD reports
`OATH CHAIN step/total`; the queue clears on phase transition and stagger. Authored touch windup
floors remain 300 ms for a fresh tell and 240 ms for a swipe follow-up.

The deterministic acceptance packet forces Oath V through
`volley 1/3 → charge 2/3 → swipe 3/3`. Measured and Journey retain an empty chain queue.

## Player combo feedback

The v2.11.2 two-entry light-input queue remains authoritative. v2.12 adds a separate visual record
of **connected** light hits:

- first contact: `CHAIN 1/3 ◆◇◇` for 1.05 s;
- second contact: `CHAIN 2/3 ◆◆◇` for 1.05 s;
- finisher contact: `LIGHT FINISHER ◆◆◆` for 1.45 s.

The feedback is screen-space and transient, above the combat centerline and away from the phone
action cluster. It does not alter damage, stamina, lunge, timing, hit-stop, input priority, or the
light-only SFX identity. Three rapid phone taps still resolve as steps `0, 1, 2`; the follow-up
queue is empty after the string.

## Deliberately not shipped

- No phase checkpoint or continuation. The one-tap retry has no runback; a checkpoint would remove
  the whole-fight endurance arc and add terminal-state complexity.
- No hidden death-responsive tuning. Death may surface an offer, but rules never change silently.
- No separate assist dashboard. The current recommended preset and one-step death offer keep the
  first-run choice small.
- No smaller touch targets, reduced input buffer, concealed tell, or reaction floor below the
  established mobile contract.
- No automatic unlock gate on Oaths. Existing players can still choose every path immediately.

Unscored phase practice remains a possible later feature only if fresh human testing shows that
late-phase repetition—not understanding or execution—is the main beginner drop-off.

## Known limits

The browser gate verifies input, state, timing, display text, audio routing, and event identity. It
cannot judge subjective difficulty, musical taste, physical speaker timbre, real vibration, or
the iOS Safari URL-bar/safe-area combination. The numerical Journey candidate is therefore
verified as coherent and transparent, not declared universally optimal. Real beginner,
intermediate, and expert cohorts remain the right source for later balance tuning.

## Changed from v2.11.2

- First-time path changes from Measured 0 to disclosed Journey -2; legacy saves remain Measured.
- Negative Grace adds named tells and recovery room; repeated deaths may offer one explicit step
  toward Grace.
- Positive paths become Oaths with authored attack packets and recovery pressure.
- The intended perfect-dodge/poise/stagger loop is taught in the game and persists once learned.
- Death feedback becomes attack-specific.
- Connected light-combo progress becomes visible.
- Score/save schema changes from v3 to v4 with backward-compatible execution fields.
- Boss health, phases, individual attack mechanics, player damage/timing, music, action SFX,
  procedural character art, collision, touch targets, input buffer, and mobile tell floors remain
  otherwise unchanged.
