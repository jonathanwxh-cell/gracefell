# GRACEFELL v2.13 release record

## Outcome

GRACEFELL v2.13 lets players pause and resume any active fight without advancing combat or audio.

- Phone and desktop expose one persistent PAUSE/RESUME button.
- The button is a 76×44px semantic DOM control beside SOUND and respects safe-area insets.
- Desktop also supports P and Escape.
- A centered pause card keeps the frozen arena visible and explains how to resume.
- Combat time, player, boss, hazards, RAF, streamed score, and Web Audio all stop together.
- Manual pause remains owned across browser focus and visibility changes.
- Attacks or dodges pressed while paused are discarded before the first resumed frame.

No boss behavior, difficulty value, attack pattern, collision rule, player timing/damage, save
schema, score, music asset, SFX identity, combat-touch geometry, runtime art asset, particle system,
or per-frame rendering workload changed.

## Design and root cause

The engine already had an interruption-safe pause path for hidden tabs and focused semantic
settings. Players could not invoke it deliberately, and collapsing all pause causes into one
boolean would let one cause incorrectly resume another.

v2.13 adds `manualPaused` as a third owner. `syncPauseState()` pauses while manual, interruption, or
settings-focus ownership remains. Both the DOM control and P/Escape call `togglePause()`. Resume
clears paused combat input before restarting with a fresh timestamp.

The semantic control remains live while RAF is stopped. It returns focus to the canvas after every
activation and publishes immediate engine-to-React state changes, avoiding a stale RESUME label.
