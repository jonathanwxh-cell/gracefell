# Gracefell v2.20 — fixed-pool phase weather

Date: 2026-07-24
Implementation: Codex (GPT-5 / Codex Desktop)
Status: merged, deployed, and publicly verified

## Outcome

Gracefell's atmosphere now develops with Malakar's three phases without adding a
runtime asset, particle pool, shader, fog surface, audio loop, dependency,
network request, save field, or gameplay modifier:

1. **Quiet Ash** — restrained vertical ash and steady grace rays.
2. **Ember Gale** — warm left-to-right streaks, wind-driven embers, stronger
   ray sway, and a warmer room.
3. **Gracefall Storm** — reversed pale wind, longer streaks, weakened rays, and
   a darker/cooler room.

Each change crossfades over 2.4 seconds with the existing phase transition.

## Cost-optimal implementation

The original 64 ash/grace motes remain the entire weather pool:

- 48 low-alpha background motes are batched into two Canvas paths and drawn
  above the cached arena floor but below telegraphs, characters, hazards, and
  world effects;
- 16 existing motes provide edge-only foreground depth below the HUD and touch
  controls;
- the existing ember emitter receives the same wind vector;
- the existing backdrop and three god rays interpolate their phase palette,
  opacity, and sway.

Motion uses deterministic sine gusts and `Game.time`. There is no weather timer,
per-frame particle creation, or random visual jitter. MENU, manual pause,
hit-stop, and browser interruption therefore freeze weather with combat.

The design follows current Canvas guidance to batch state changes and reuse
existing work rather than add expensive blur or oversized offscreen layers:

- [MDN: Optimizing canvas](https://developer.mozilla.org/en-US/docs/Web/API/Canvas_API/Tutorial/Optimizing_canvas)
- [web.dev: Improving HTML5 Canvas performance](https://web.dev/articles/canvas-performance)
- [MDN: Long animation frame timing](https://developer.mozilla.org/en-US/docs/Web/API/Performance_API/Long_animation_frame_timing)

## Readability and comfort

- The central 42% screen-space band excludes foreground ash.
- Combat layers always render above background ash.
- HUD, MENU, MIX, PAUSE, SOUND, and touch actions render above every weather
  layer.
- The reserved hazard red is not used by weather.
- Flash-reduced or shake-disabled play lowers motion to 45% and streak length
  to 55%.
- No lightning, full-screen fog, added camera motion, or continuous wind sound
  was introduced.

The desktop gate verifies:

The final post-documentation same-scene weather render probe recorded:

| Profile | Median | p95 |
|---|---:|---:|
| Quiet Ash | 0.5 ms | 1.0 ms |
| Ember Gale | 0.4 ms | 0.7 ms |
| Gracefall Storm | 0.4 ms | 0.6 ms |

An earlier green full run measured Quiet Ash at 0.4/0.6 ms and Gracefall Storm
at 0.4/1.0 ms. Across both runs, the worst Phase 3 regression was therefore
0 ms at the median and 0.4 ms at p95, inside the 0.5/1.0 ms feature budgets.

The complete Playwright pass also retained every earlier desktop, 390×844
mobile, and true-touch combat, difficulty, death/retry, victory, score-history,
navigation, music, MIX, audio-pressure, accessibility, and responsive assertion.

A separate persistent-profile probe recorded a complete victory, closed the
Chromium process, reopened it with the same browser profile, and recovered the
identical grade, time, damage, wounds, dodges, flask use, attempt number, and
completion timestamp. This confirms session persistence on the same
browser/device/origin, in addition to the suite's save round-trip and score
chronicle checks.

- Phase 3 reverses into longer pale streaks.
- The arena ring tell, player, boss, HUD, and controls remain dominant.
- Phone weather is visible without covering the central fight or thumb actions.

The accepted pre-deployment artifacts remain in the platform temporary
`gracefell-qa-v220-predeploy` directory. A separate full production replay was

## Changed from v2.19

- Atmospheric rendering now responds to boss phase.
- The existing fixed mote pool is depth-layered and batched.
- Existing embers, backdrop, and rays use the same phase weather profile.
- Runtime assets, audio graph, difficulty, boss actions, player timing/damage,
  save schema v6, touch geometry, and score logic are unchanged.
